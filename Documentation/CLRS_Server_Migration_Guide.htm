7. Maintenance Procedures

7.1 Adding a New Environment

Because URLs now use the dynamic #This_Server# variable (populated from CGI.SERVER_NAME), deploying to a new server requires no URL changes. Simply:

  1. Deploy the application files to the new server.
  2. Ensure the CFFILE_RFS_Destination_Dir physical path exists and has appropriate write permissions.
  3. If MFA should be bypassed on the new server, add the server name to the exclusion list in MfaCookiecheck.cfm.
  4. If a default test user is needed, add a cfelseif block in Admin.full.cfm, setTestUser.cfm, and UpdateTestUser.Action.cfm.

7.2 File Upload Directory Permissions

The upload destination directory must have write permissions for the ColdFusion service account:

  • Path: D:\web\inetpub\wwwroot2\ClientService\DocUploadsFromCF2018\Doc.ContingentLiabilities\RequestForSubmissions\
  • Required: Read/Write for the ColdFusion application pool identity

7.3 Source Control

All code changes are checked into the GitHub repository. Changes to server configuration or paths should be committed with descriptive messages referencing the migration.
